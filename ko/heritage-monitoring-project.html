국가유산 모니터링·조사·등록 시스템

국가유산청 Open API, AI 손상 탐지, Firebase 실시간 동기화를 통합한 크로스플랫폼 현장 조사 시스템.

이 프로젝트는 문화재 현장 조사 과정의 디지털 전환(Digital Transformation)을 목표로, 기존 종이 기반 조사 방식을 AI 기반 자동 분석·클라우드 연동 중심의 통합 조사 시스템으로 재설계한 프로젝트이다. Flutter(Web/Android/iOS) 기반 프론트엔드와 FastAPI 백엔드, PyTorch 손상 탐지 모델, Firebase 실시간 데이터베이스를 하나의 워크플로우로 통합하여 구축하였다.

FastAPI 서버는 국가유산청 Open API의 XML 데이터를 JSON 구조로 변환하여, 유산명·종목·지역 기반의 다중 조건 검색과 페이지네이션 기능을 제공한다. 각각의 문화유산 상세 화면에서는 지정 현황, 소유자/관리자 정보, 소재지, 좌표 등 필수 정보를 확인할 수 있으며, 사용자가 직접 국가유산을 수동 등록할 수도 있다.

핵심 기능은 AI 기반 손상부 조사 모듈이다. 현장에서 촬영한 이미지를 앱에서 업로드하면, 백엔드의 PyTorch DETR 모델이 이를 분석하여 네 종류의 손상 유형(갈램, 균열, 부후, 압괴/터짐)을 바운딩 박스와 신뢰도 점수와 함께 반환한다. 또한 탐지 결과를 기반으로 A~D 등급의 자동 심각도(Severity Grade)를 산정하여 조사 효율성을 크게 높였다. 분석 결과와 조사자의 상세 의견, 손상 위치·현상 등은 모두 Firestore에 실시간으로 저장된다.

시스템은 현장 조사 프로세스를 3단계 탭 구조로 구성했다:

  • 현장 조사: 기본 정보, 메타 정보, 현황 사진, AI 기반 손상 조사
  • 조사자 의견: 보존관리 이력, 조사 의견, 관리 사항, 수정 이력
  • 종합 진단: 손상 종합 분석, 등급 분류, AI 예측 결과

Firebase Firestore 스트림을 기반으로 모든 수정 사항이 즉시 UI에 반영되며, Storage에 업로드된 이미지 역시 실시간 갱신된다. 또한 모든 편집 기록은 수정 이력 로그로 자동 저장되어 향후 감사(audit)나 이력 추적이 가능하다.

본 시스템은 AI 모델·실시간 데이터베이스·크로스플랫폼 UI·REST API 파이프라인을 하나의 현장 업무 시스템으로 통합한 사례로, 문화유산 보존 관리 업무의 디지털 전환을 위한 실제 운영 수준의 기반을 구축하였다. 향후에는 손상 지도(Spatial Damage Map), 오프라인 모드, 다국어 UI 등으로 확장할 수 있는 구조를 갖추고 있다.

기술 스택: Flutter, Dart, Firebase(Firestore/Storage), FastAPI, Python, PyTorch(DETR), Docker, Nginx, XML→JSON Proxy
