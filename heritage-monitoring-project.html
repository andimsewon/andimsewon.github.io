Heritage Monitoring and Registration System

A cross-platform field-survey system integrating national heritage data, AI-based damage detection, and real-time cloud synchronization.

This project focuses on modernizing Korea's cultural heritage field-survey workflows by transitioning paper-based inspection records into a fully digital, AI-assisted monitoring platform. Built with Flutter (Web/Android/iOS) and a FastAPI backend, the system integrates the National Heritage Service Open API, PyTorch-based damage detection, and Firebase real-time databases to support complete end-to-end survey and registration tasks.

The system retrieves official heritage records through a FastAPI proxy that converts XML data from the National Heritage Service into structured JSON, enabling multi-criteria search (heritage type, region, keyword) with infinite pagination. Each heritage asset provides detailed information including designation category, ownership, management agency, official address, and geographic coordinates.

A key feature is the AI-powered damage survey module, which leverages a custom DETR-based PyTorch model to detect four types of structural deterioration‚Äîcracks, splits, biological decay, and crushing/burst damage. Images captured on-site are uploaded through the Flutter app, processed by the backend AI pipeline, and returned with bounding boxes, class labels, confidence scores, and an automatically assigned severity grade (A‚ÄìD). These results are stored in Firebase along with survey metadata such as component name, location, inspector notes, and timestamps.

The platform includes a structured three-tab workflow for field inspectors:

  • Field Survey: basic info, metadata, site photos, and damage detection
  • Inspector Notes: preservation history, survey results, and management actions
  • Comprehensive Assessment: aggregated damage analysis, severity classification, and AI-assisted insights

All updates are synchronized in real time using Firestore streams, ensuring consistent multi-device collaboration during field inspections. Photos and damage records are stored in Firebase Storage, and edit histories are automatically logged for auditability.

The system demonstrates practical integration of cross-platform UI, REST API pipelines, real-time cloud data synchronization, and custom deep-learning inference inside a production-ready workflow. It showcases the feasibility of scalable digital transformation for cultural-heritage management and lays the foundation for future extensions such as GIS-based damage maps, offline-first mobile modes, and multi-language interfaces.

Tech Stack: Flutter, Dart, Firebase (Firestore/Storage), FastAPI, Python, PyTorch, DETR, Docker, Nginx, XML‚ÜíJSON proxy, Responsive Web Architecture
