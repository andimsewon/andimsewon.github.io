This project was conducted under the Korean AI Square Summer Program at Purdue University, focusing on developing a cost-effective, scalable, and dual-scope IoT monitoring system for smart beekeeping.

Traditional smart hives mainly focus on internal temperature and humidity while relying on expensive GSM/LTE modules, which limit scalability and real-time monitoring. To address these constraints, WaggleNet integrates Wi-Fi LoRa 32 (V3) modules to collect and transmit both internal and external environmental data using a LoRaâ€“MQTT hybrid architecture.

Each worker nodeâ€”equipped with DHT22, LDR, and GPS sensorsâ€”collects environmental data every three minutes and transmits it to a master node via LoRa. The master node appends a UTC timestamp, then forwards the structured JSON packets to an MQTT server, which relays the data to a cloud database (SQLite). A Flutter-based mobile application visualizes the data in real time, displaying hive temperature, humidity, light intensity, and location. It also issues emergency alerts through Firebase Cloud Messaging when critical thresholds are exceeded.

Field tests were conducted at Purdue University's KSW building and a real bee yard near South 525 West, Indiana, confirming the system's robustness and scalability. LoRa communication remained stable up to 110 meters in open areas and 95 meters in obstructed environments, validating its reliability even inside wooden hive structures.

Future work will include AI-based anomaly detection, hardware miniaturization, and adaptive routing protocols to enhance intelligence and autonomy in field operations.
